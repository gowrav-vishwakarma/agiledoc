Directory Structre Of Agiletoolkit

The following is the base directory structure for agiletoolkit project. The structure shown here is from your project_root. For simplicity and to keep learning track faster, we are not taking every file and folder in details here. We will cover the remaining files and folders later in the course.

  • atk4

    This is the main framework. You MUST not change any thing in here.

  • atk4-addons

    This is extenssion to main framework. Various utilitites and extra features are managed in atk4-addons.

  • lib

    This is your library folder. Before searching for any class in atk4 main framework the files and classes are searched here first. So here you will create your Models, View, Controllers or any class you want to add to the system. The following are main files and folders in this folder:

    • Frontend.php

      This is main application class. your main index file will create object of this class. Here you will define the locations to search for various files, database connectivity, Menus etc.

    • Model

      This folder will contain your ORM Models.

  • page

    This folder will contain your main web pages.

  • templates

    This folder will contain your html part. This folder holds template for your complete website as wellas each indivisual visual element. This folder is searched for every required template first, if file is not found here, it is searched in main atk4 framework's template folder.

  • index.php

    This is main page that is called for any request to server. This page usually includes the framework bootstrap file and then creates application object defined in lib/Frontend.php file.

  • config-default.php

    This is the default page, which holds basic settings and configuration.
